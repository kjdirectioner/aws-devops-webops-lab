## 🧩 Overview

Ansible is used here for two jobs:

1. deploy Nginx and website content
2. deploy the monitoring stack

The project now supports three inventory workflows:

- manual inventory using `inventory.example.ini` (legacy, public IP)
- Terraform-generated inventory from `terraform/` (legacy, public IP)
- Terraform-generated inventory from `terraform-modular/` (current, private subnet via EICE)

> **Note:** the private-subnet instance introduced in Phase 5/6 has no public or reachable private IP from the operator's machine. Instead of an IP, the generated inventory for that layout uses the **instance ID** as `ansible_host`, because EC2 Instance Connect Endpoint's `send-ssh-public-key` and `open-tunnel` calls key off instance ID, not IP. `ansible-project/ansible.cfg`'s `ProxyCommand` opens that EICE tunnel automatically per host — this wiring is now complete and validated end-to-end (see [06 — Terraform Modular Refactor](./6-terraform-modular.md)).

## 🗂️ Files Used

| File | Purpose |
|------|---------|
| `ansible-project/playbook.yml` | Deploys Nginx to `web_servers` |
| `ansible-project/monitoring.yml` | Deploys Node Exporter, Prometheus, and Grafana |
| `ansible-project/inventory.example.ini` | Safe template for manual runs (legacy, public IP) |
| `ansible-project/inventory.generated.ini` | Generated by `terraform/` (legacy, public IP) |
| `ansible-project/inventory.ini` | Generated by `terraform-modular/` (current, instance ID + EICE) |
| `ansible-project/ansible.cfg` | Defines the EICE `ProxyCommand` used to reach the private-subnet instance |

## ⚙️ Inventory Structure

Legacy (public-IP) layout — both groups use a reachable host/IP:

```ini
[web_servers]
<ubuntu_host_or_ip> ansible_user=ubuntu ansible_ssh_private_key_file=~/.ssh/<ubuntu_key>.pem

[monitoring]
<ubuntu_host_or_ip> ansible_user=ubuntu ansible_ssh_private_key_file=~/.ssh/<ubuntu_key>.pem
```

Current (private-subnet) layout — both groups use the instance ID, since there is no IP to target directly:

```ini
[web_servers]
web-server-1 ansible_host=<instance-id>

[monitoring]
monitor-1 ansible_host=<instance-id>
```

`web_servers` is used for web and exporter tasks.

`monitoring` is used for Prometheus and Grafana.

In the current single-instance setup, both groups point to the same Ubuntu host either way.

## 🚀 Run The Playbooks

### Manual inventory (legacy, public IP)

```bash
cd ansible-project
cp inventory.example.ini inventory.ini
# edit host IP and SSH key path
ansible-playbook -i inventory.ini playbook.yml
ansible-playbook -i inventory.ini monitoring.yml
```

### Terraform-generated inventory (legacy, public IP, from `terraform/`)

```bash
ansible-playbook -i ansible-project/inventory.generated.ini ansible-project/playbook.yml
ansible-playbook -i ansible-project/inventory.generated.ini ansible-project/monitoring.yml
```

### Terraform-generated inventory (current, private subnet, from `terraform-modular/`)

```bash
ansible-playbook -i ansible-project/inventory.ini ansible-project/playbook.yml
ansible-playbook -i ansible-project/inventory.ini ansible-project/monitoring.yml
```

No manual tunnel setup is required — `ansible.cfg`'s `ProxyCommand` sends a temporary SSH public key to the instance and opens the EICE tunnel automatically for each connection.

## 🛠️ What Each Playbook Does

### `playbook.yml`

- installs Nginx on Ubuntu
- copies `index.html` into `/var/www/html/`
- ensures the Nginx service is enabled and running

### `monitoring.yml`

- installs Node Exporter on `web_servers`
- installs Prometheus on `monitoring`
- installs Grafana on `monitoring`

## ✅ Validation

```bash
ansible-playbook --syntax-check -i ansible-project/inventory.example.ini ansible-project/playbook.yml
ansible-playbook --syntax-check -i ansible-project/inventory.example.ini ansible-project/monitoring.yml
ansible all -i ansible-project/inventory.ini -m ping
```

Expected results:

- Ansible connects successfully (over the EICE tunnel for the current layout)
- Nginx serves the custom page
- Node Exporter listens on port `9100`
- Prometheus runs on port `9090`
- Grafana runs on port `3000`
